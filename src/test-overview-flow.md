# 概览页引导流程测试

## 信息架构（重构后）

- **激活状态卡**：三步（Step 1 工具可用 / Step 2 供应商已配置 / Step 3 开始使用），每步有完成态与说明。
- **下一步卡**：单一 Primary CTA + 最多 2 个 Secondary；阻塞时展示「阻塞原因」与「建议动作」；CLI 未安装时仅展示安装区块，不展示便捷入口。
- **便捷入口**：CLI 已安装后按能力分组展示（配置 / 扩展 / 集成 / 使用）；OpenClaw 单独为工作区、环境变量、工具权限、Agents 配置。

## 测试场景（手动回归）

### 场景1：CLI 未安装 + 无供应商
- [ ] 左侧「激活状态」：Step 1 未勾选（CLI: 未安装），Step 2/3 未勾选。
- [ ] 下一步卡：显示「阻塞原因：未检测到 CLI」「建议：先安装 CLI…」；仅展示 CLI 安装区块（开始安装 / 手动安装 / 刷新）；不展示便捷入口。

### 场景2：CLI 已安装 + 无供应商
- [ ] 激活状态：Step 1 已勾选，Step 2/3 未勾选。
- [ ] 下一步卡：阻塞原因「尚未配置供应商」；Primary「快速导入」，Secondary「管理供应商」；展示便捷入口（配置 / 扩展 / 集成 / 使用）。

### 场景3：CLI 已安装 + 有供应商
- [ ] 激活状态：三步均勾选（或 Step 3 文案为「开始使用」）。
- [ ] 下一步卡：无阻塞文案；Primary 为「打开会话」（支持会话的应用）或「管理供应商」；Secondary 为「管理供应商」（当有会话时）；便捷入口正常。

### 场景4：OpenClaw（有/无供应商）
- [ ] 激活状态：三步逻辑同通用应用。
- [ ] 下一步：有供应商时为「配置环境变量」+「管理供应商」；便捷入口为工作区 / 环境变量 / 工具权限 / Agents 配置。

### 场景5：Qwen Node 缺失或版本过低
- [ ] 选择 QwenCode 且 CLI 未就绪时，安装区块内展示 Node 缺失或版本过低提示（与现有一致）。

## 状态机验证

```
进入概览 -> 检测 CLI/环境
  否 -> 显示安装引导 + 安装操作（不展示便捷入口）
  是 -> 检测供应商
    否 -> 显示配置引导 + 快速导入/管理供应商 + 便捷入口
    是 -> 根据应用能力给主入口（打开会话 / 配置 Env / 管理供应商）
```
